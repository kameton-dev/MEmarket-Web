# Создание и дальнейшее управление репозиторий с помощью memarketrepo

**memarketrepo** — это специализированная консольная утилита на языке Python, предназначенная для автоматизации создания, настройки и обновления структуры децентрализованных репозиториев Millennium Market (MEmarket).

Скачать можно на [странице скачивания](../download/index.html) во вкладке "утилиты"

## Принцип работы и быстрый старт

Утилита позволяет развернуть и подготовить репозиторий к публикации всего за несколько шагов через терминал.

### Шаг 1: Инициализация репозитория
Создайте пустую директорию для вашего будущего репозитория, откройте в ней терминал и выполните команду:

```bash
memarketrepo init
```

Утилита в интерактивном режиме запросит название репозитория и имя автора, после чего автоматически создаст:

- Пустой файл индекса index.json.

- Служебную директорию metadata/ для хранения файлов приложений.

### Шаг 2: Добавление приложений

Внутри созданной директории metadata/ создайте подпапку для вашего приложения (например, metadata/vasya_pupkin/) и скопируйте туда установочный пакет (.xap, .appx или .appxbundle).

### Шаг 3: Настройка адреса публикации

Укажите базовый URL-адрес (домен), по которому ваш репозиторий будет доступен в сети. Это необходимо для корректного формирования ссылок на скачивание файлов клиентами:

```bash
memarketrepo seturl http://ваш-домен.ru
```

### Шаг 4: Генерация и обновление индексов

Для автоматической сборки репозитория выполните команду:

```bash
memarketrepo update
```
Что происходит на этом этапе:

1. Утилита сканирует директорию metadata/ на наличие подпапок с приложениями.

2. Обнаружив установочный пакет, скрипт анализирует его структуру как ZIP-архив и парсит внутренний манифест приложения для извлечения системных данных (версии, идентификаторы и т.д.).

3. В папке каждого приложения генерируется локальный манифест entry.json со всеми подробностями.

4. Создается или обновляется глобальный файл index.json, который мобильный клиент использует для вывода общего списка доступного софта.

## Дополнительные файлы оформления

Чтобы приложения в каталоге выглядели информативно и красиво, вы можете вручную добавить в папку конкретного приложения следующие файлы (например, внутрь metadata/vasya_pupkin/):

### 1. Описание (description.md)

Текстовый файл в формате Markdown для подробного описания возможностей приложения. Сюда можно скопировать содержимое файла README из исходного кода. При запуске команды update утилита автоматически подтянет этот текст в карточку приложения.

### 2. Дополнительные метаданные (metadata.yaml)

Файл конфигурации для указания информации, которую невозможно автоматически извлечь из манифеста .xap или .appx.

Доступные поля для заполнения:

```yaml
title: "Название приложения" # Используется, если имя из манифеста отображается некорректно
creator: "Имя автора или команды разработчиков"
summary: "Краткое описание приложения в одну строчку"
category: "tools"
```

**ВАЖНО!** в строке "category" можно писать любые значения, но другие категории кроме из [списка сортируемых категорий](#categories) будут отправлться в общую - **"Прочее"**.

### 3. Папка со скриншотами (screenshots/)

Директория, внутрь которой можно загрузить изображения интерфейса приложения для отображения в галерее мобильного клиента.

## Важные нюансы и ограничения

1. **Иконки для .XAP (Windows Phone 8.1):** На текущем этапе утилита не поддерживает автоматическое извлечение изображений иконок из пакетов .xap. Вам необходимо вручную добавить файл иконки в папку приложения и обязательно переименовать его в icon.расширение (например, icon.png).

2. **Статус бета-версии:** Скрипт находится на ранней стадии разработки и в редких случаях может работать нестабильно. Рекомендуется вручную проверять корректность сформированных файлов entry.json и index.json после выполнения обновлений. При необходимости их можно отредактировать в любом текстовом редакторе.

3. **Простота использования:** Архитектура утилиты специально проектировалась так, чтобы минимизировать количество действий. Создание собственного независимого магазина приложений доступно любому пользователю и не требует глубоких навыков администрирования.
